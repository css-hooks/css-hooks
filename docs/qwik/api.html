    API

    @css-hooks/qwik

    createHooks: (_: Config, _?: Options) => [StyleSheet, CssFn]

    Given a hook configuration and (optional) options, this function produces a static style sheet, which must be added to the document, along with a css function that can leverage the configured hooks.

    type Config = Record<HookName, HookSpec>

    This configuration object defines the hooks available for use in the css function.

    type HookName = string

    The alias used to reference the hook in the css function

    type HookSpec

    A typical HookSpec is a specific type of string that defines the condition under which a hook applies, following one of these formats:

    1. ":<pseudo>", where <pseudo> is any pseudo-class such as hover, active, or focus.
    2. "<selector>", where <selector> must contain an & character functioning as a placeholder for a base selector. For example, a value of ":checked + &" would produce a hook that is activated whenever the element is immediately preceded by a :checked element.
    3. "@container <query>", where <query> can be replaced with any container query condition.
    4. "@media <query>", where <query> can be replaced with any media query condition.

    Two additional types of HookSpec allow you to use boolean and/or logic to define the conditions under which the the hook should apply. These are defined using an object:

    1. { and: HookSpec[] }
    2. { or: HookSpec[] }

    type Options = { ... }

    Options that affect how CSS content is rendered

    1. debug?: boolean   When enabled, uses human-readable CSS variable names.
    2. fallback?: "unset" | "revert-layer"   Which fallback keyword to use. "revert-layer" is the best choice, but it's highly experimental.

    type StyleSheet = string

    A string of CSS content that must be added to the document to support the configured hooks

    type CssFn = (_: WithHooks<CSSProperties>) => CSSProperties

    The css function accepts an object consisting of CSS properties and nested style objects, flattening it to produce a properties object compatible with the style prop.

    @css-hooks/recommended

    Note: Requires TypeScript version 5.3 or later.

    recommended: (_: Config) => HooksConfig

    Produces a hook configuration with a default set of hooks to address the most common use cases.

    type Config = { ... }

    Use these settings to control which hooks to generate and their specific details.

    1. breakpoints?: string[]   A list of breakpoints used to produce width media queries, e.g. ["500px", "1000px"]. Note that these must be specified in ascending order.
    2. colorSchemes?: ("dark" | "light")[]   A list of color schemes used to produce prefers-color-scheme media queries.
    3. pseudoClasses?: ":<pseudo>"[]   A list of pseudo-classes you want to use, e.g. [":hover", ":active", ":nth-child(odd)"]

    type HooksConfig = Parameters<typeof createHooks>[0]

    The hook configuration object, which you pass to the createHooks function to create the recommended hooks.
